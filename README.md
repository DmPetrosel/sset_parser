# SSET Parser — Telegram-парсер заказов

Многопользовательский Telegram-бот, который мониторит чаты и уведомляет пользователей о сообщениях, похожих на заказы. Фильтрация работает на основе морфологического анализа ключевых слов (русский язык) + паттернов заказов — без AI.

## Возможности

- Мониторинг всех групп и каналов, в которых состоит пользователь
- Морфологический анализ через `pymorphy3`: "заказываю", "заказов", "заказал" — всё совпадёт с ключевым словом "заказ"
- Поддержка фраз из нескольких слов в ключевых словах
- Широкий набор паттернов заказов: ищу, нужен, бюджет, оплата, срочно и т.д.
- Стоп-слова для исключения нежелательных сообщений
- Настройка минимального количества совпадений ключевых слов
- Авторизация через QR-код (поддержка 2FA)
- Многопользовательский режим: у каждого свои настройки и сессия
- Автозапуск сессий при перезапуске
- Деплой через Docker

## Установка и запуск

### Через Docker (рекомендуется)

**1. Клонировать репозиторий**

```bash
git clone <repo-url>
cd sset_parser
```

**2. Создать `.env` файл**

```env
BOT_TOKEN=токен_от_BotFather
GIGACHAT_CREDENTIALS=credentials_от_GigaChat  # опционально
```

**3. Получить Telegram API credentials**

Зайдите на [my.telegram.org](https://my.telegram.org), создайте приложение и получите `api_id` и `api_hash`. Вставьте их в `sset_parser/config.py`:

```python
MASTER_API_ID = ваш_api_id
MASTER_API_HASH = "ваш_api_hash"
```

**4. Запустить**

```bash
docker-compose up -d
```

**5. Посмотреть логи**

```bash
docker-compose logs -f
```

---

### Локальный запуск (без Docker)

**1. Установить зависимости**

```bash
pip install poetry
poetry install
```

**2. Создать папку для данных**

```bash
mkdir -p data/sessions
```

**3. Задать переменные окружения**

```bash
export BOT_TOKEN=...
export GIGACHAT_CREDENTIALS=...
```

**4. Запустить**

```bash
cd sset_parser
python main.py
```

## Команды бота

| Команда | Описание |
|---|---|
| `/login_qr` | Авторизация Telegram-аккаунта через QR-код |
| `/settings` | Настройка ключевых слов, стоп-слов и порога совпадений |
| `/status` | Проверить, активен ли парсер |

## Настройка через `/settings`

1. **Ключевые слова** — слова через запятую, по которым ищем сообщения. Поддерживаются фразы: `нужен дизайнер, логотип, фирменный стиль`
2. **Стоп-слова** — слова через запятую, при наличии которых сообщение игнорируется: `продаю, отдам, реклама`
3. **Минимум совпадений** — сколько ключевых слов должно встретиться в одном сообщении (1, 2, 3...)

Сообщение попадает в уведомление если:
- Не содержит стоп-слов
- Совпало **не менее N** ключевых слов (с учётом морфологии)
- Содержит хотя бы один признак заказа: `ищу`, `нужен`, `бюджет`, `оплата`, `срочно` и т.д.

## Переменные окружения

| Переменная | Описание |
|---|---|
| `BOT_TOKEN` | Токен Telegram-бота от @BotFather |
| `GIGACHAT_CREDENTIALS` | Credentials GigaChat API (опционально) |
